Getting Started - Part 1

Configuration

The following are the instructions for setting up automatic emails sent in response to registrations on our blog and new blog posts.

Please visit our blog for the latest changes to blog posts.

Generate a GitHub OAuth Token

Generator a GitHub OAuth Token to allow your blog to connect to your GitHub repository to send blog posts as emails when you push a new blog post.

  1. Login to your GitHub account and navigate to Profile > Settings > Developer Settings > Personal access tokens > Tokens (classic).

  2. Click the Generate new token button and generate a classic token.

  3. Give the token the repo scope.

  4. Copy the generated token and save it for later.

Create a GMail App Password

Create a GMail App Password to allow your blog to send registration and blog post emails to your users.

  1. Login to your Gmail account, click your profile icon, then click Manage Your Google Account.

  2. In your Google Account, click on Security.

  3. Under How you sign in to Google, click 2-Step Verification.

  4. Click on App Passwords.

  5. Generate a new app password for Mail with the device <name of your blog>.

  6. Copy the generated password and save it for later.

Blog Settings

Using the GitHub token and the GMail App Password created above, fill in the settings for your blog.

  1. Navigate to the blog-service Google Firestore database.

  2. Under settings, click on the template document.

  3. Click the 3-dot menu to the far right and select Add similar document.

  4. Make the Document ID your domain name.

  5. Modify the settings for your blog.

  6. Make the origin your blog URL.

  7. Add similar document for the domain you just created and make the Document ID the path for your GitHub repository. NOTE: replace a slash with %2F
